Windows – Comment utiliser Git, Npm, Bower et Ruby Gem Manager derrière un proxy ?

L’utilisation de gestionnaires de paquets derrière un proxy nécessite un ensemble de configuration propres à chaque outils. Voici un ensemble non exhaustif de solutions permettant de configurer les gestionnaires de paquets NPM, Bower et les outils dont ils peuvent dépendre : Git et Ruby Gem.

Celle liste n’est pas définitive et peut être sujette à modifications. Si vous avez de meilleures solutions ou d’autres logiciels à configurer, n’hésitez pas !

CNTLM

CNTLM est un proxy local qui va simplifier la configuration des autres outils. Il permet de créer une redirection paramétrable sans mot de passe qui pourra gérer les entrées vers le proxy nécessitant une authentification.

En bref, CNTLM permet de centraliser la configuration du proxy en précisant identifiant et mot de passe, sans que ce dernier soit stocké en clair.

Npm

Npm est le plus gros gestionnaire de paquet Javascript dédié notamment aux modules Node.js. Il permet d’automatiser les processus d’installation, de mise à jour et de vérification des dépendances entre différents modules. Son utilisation derrière un proxy nécessite une configuration préalable.

Npm peut dépendre de l’utilisation de Git. Une section de cet article est dédié à sa configuration.

Npm possède un fichier de configuration global présent à l’adresse :

C:\Users\%Username%\.npmrc

Il existe deux méthodes pour configurer l’outil :

Configurer Npm en lignes de commande

Insérez ces lignes de commande directement depuis votre console : 

npm config set proxy http://<userName>:<password>@<adresseDuProxy>:<port>
npm config set https-proxy http://<userName>:<password>@<adresseDuProxy>:<port>
npm config set registry http://registry.npmjs.org/

Configurer Npm depuis son fichier de configuration

Directement dans le fichier de configuration, ajouter les lignes suivantes :

registry=http://registry.npmjs.org,
proxy=http://<userName>:<password>@<adresseDuProxy>:<port>,
https-proxy=http://<userName>:<password>@<adresseDuProxy>:<port>,

Si l’identification SSL ne s’avère par nécessaire, ajouter la ligne suivante :

strict-ssl=false /* Si l’identification SSL n’est pas nécessaire */
N.B. Les identifiants ne sont à préciser que lorsqu’ils sont nécessaires. Je vous conseille vivement de ne pas les préciser en clair dans vos fichier et d’utiliser à la place l’outil CNTLM.

Références

  • npm-config, https://docs.npmjs.com/misc/config, *Npm, 21.01.2016

Bower

Comme NPM, Bower est un gestionnaire de paquet nécessitant d’être configurer derrière un proxy. La configuration de GIT (voir section dédiée) est obligatoire pour utiliser Bower.

Bower possède deux types de fichiers de configuration, un global présent à l’adresse :

C:\Users\%Username%\.bowerrc

Et un propre à chaque projet, situé dans le dossier contenant le fichier bower.json., portant également le nom .bowerrc. Le fichier local est facultatif. Les options qu’ils précisent surchargent celle du fichier global.

Configuration de Bower depuis le fichier de configuration

Au sein du fichier de configuration ajouter les lignes suivantes : 

{
     …
    “proxy”: “http://<userName>:<password>@<adresseDuProxy>:<port>”,
    “https-proxy”: “http://<userName>:<password>@<adresseDuProxy>:<port>”,
    “strict-ssl”: false /* Si l’identification SSL n’est pas nécessaire */
}
N.B. Les parties vertes sont requises en cas d’identification obligatoire au proxy. Il n’est pas conseillé de conserver un mot de passe en clair. Favoriser pour cela l’utilisation d’outils comme CNTLM (voir plus haut).

Références

- Bower Configuration, http://bower.io/docs/config/, Bower,

GIT

La portée de configuration de GIT se définit soit localement, soit globalement. Pour la gestion du proxy, favoriser une configuration globale propre au système.
Les lignes de commande vont automatiquement modifier le fichier de configuration. Cependant il est possible d’éditer le fichier manuellement.

Configuration de Git en lignes de commande

git config –global http.proxy http://<userName>:<password>@<adresseDuProxy>:<port>
git config –global https.proxy http://<userName>:<password>@<adresseDuProxy>:<port>



N.B. Les parties vertes sont requises en cas d’identification obligatoire au proxy. Il n’est pas conseillé de conserver un mot de passe en clair. Favoriser pour cela l’utilisation d’outils comme  CNTLM (voir plus haut).

Configuration de Git depuis le fichier de configuration

Le fichier de configuration global de l’utilisateur est présent à l’adresse :
C:\Users\Username.gitconfig
Ajouter les lignes suivantes :
 [http]
    proxy = http://<userName>:<password>@<adresseDuProxy>:<port>
[https]
    proxy = http://<userName>:<password>@<adresseDuProxy>:<port>

Référence

- Git Config, Documentation, http://git-scm.com/docs/git-config

Ruby

Il n’existe pas de fichier de configuration pour l’outil d’installation des gems. L’outil utilise les variables d’environnements du système.
Pour se faire, il suffit de taper les lignes suivant dans l’invité de commande :
SET HTTP_PROXY=http://<userName>:<password>@<adresseDuProxy>:<port>
SET HTTPS_PROXY=http://<userName>:<password>@<adresseDuProxy>:<port>
